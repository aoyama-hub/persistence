Markdown

Processing Jupyter Notebook Entries
Export your localized workspace notebook tracking files via command line:

Bash
jupyter nbconvert --to markdown your-analysis.ipynb
Move the generated your-analysis.md file AND its corresponding assets asset folder your-analysis_files/ directly into the content directory:

Plaintext
src/content/blog/your-analysis.md
src/content/blog/your-analysis_files/
Ensure image references use localized paths: ![png](./your-analysis_files/output_0.png). Astro maps and scales these assets accurately during optimization compiles.

## 📋 The GitHub Settings Deployment Checklist

Follow this linear sequence exactly to initialize your remote repository cleanly:

- [ ] **Create the Remote Repository:** Go to GitHub and open a brand-new repository named exactly `persistence`. Leave it completely blank (do *not* pre-initialize with a remote README or `.gitignore`).
- [ ] **Link and Execute the First Push:** Open your terminal inside your local project workspace and run:
  ```bash
  git init -b main
  git add .
  git commit -m "feat: infrastructure initialization"
  git remote add origin https://github.com/aoyama-hub/persistence.git
  git push -u origin main
[ ] Switch Pages Build Routing Engine: Go to your GitHub repository in your web browser, navigate to Settings → Pages. Locate the Build and deployment area. Under Source, click the dropdown and change it from Deploy from a branch to GitHub Actions.

[ ] Track Execution Pipelines: Click the Actions tab along your top repository navigation bar. You should see a live deployment run executing. Click into the workflow timeline to watch the build task compile and the deployment stage pass.

[ ] Diagnostic Debugging: If a content file fails formatting validation rules (such as an incorrect date style or missing parameter), the Build Website Assets stage will turn red. Click into that step to review the explicit logs generated by Astro's schema validator.
